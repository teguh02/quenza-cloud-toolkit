{# Intro #}

Pengantar

Quenza Cloud Toolkit adalah aplikasi manajemen backup & restore data server yang sederhana, aman, dan terpusat. Anda dapat membuat project backup, memilih sumber (direktori, file, database), menjadwalkan backup otomatis, dan mengirim hasilnya ke berbagai tujuan penyimpanan.

Aplikasi memakai sistem Master Password tunggal — tidak ada registrasi pengguna. Cocok untuk penggunaan internal/terbatas.

{# Getting started #}

Memulai

  1. 1Login menggunakan Master Password yang dikonfigurasi oleh administrator (variabel MASTER_PASSWORD_HASH di .env).
  2. 2Atur Zona Waktu di menu Settings agar penjadwalan & waktu tampil sesuai lokasi Anda.
  3. 3Buat Destination (tujuan penyimpanan), lalu buat Project dan pilih destinasinya.
  4. 4Jalankan backup manual atau aktifkan jadwal otomatis.
{# Projects #}

Projects & Sumber Backup

Sebuah Project adalah satu unit konfigurasi backup. Di dalamnya Anda menentukan sumber data, format arsip, destinasi, dan jadwal.

Buka Projects → New Project, beri nama, lalu pilih format output:

  • • .zip — kompatibel luas, mudah dibuka di mana saja.
  • • .tar.gz — umum di lingkungan Linux/Unix.

Di detail project, tambahkan sumber backup secara fleksibel:

Direktori & File

Dipilih lewat File Manager. Direktori di-backup secara rekursif (semua sub-folder & file ikut).

Database MySQL & PostgreSQL

Menggunakan mysqldump / pg_dump. Hasil dump disertakan ke dalam arsip.

{# File Manager #}

Integrated File Manager

Saat menambah sumber direktori/file, gunakan File Manager bawaan untuk menjelajah filesystem server.

  • • Panel kiri menampilkan struktur folder; panel kanan menampilkan isi dengan checkbox.
  • • Gunakan breadcrumb, tombol naik level, dan refresh untuk navigasi.
  • • Centang folder/file yang ingin di-backup, lalu klik Tambahkan Sumber.
Tips: cukup centang folder induk — seluruh isinya (termasuk sub-folder bertingkat) otomatis ikut saat backup dijalankan. Isi folder dibaca ulang setiap kali backup berjalan, jadi file baru selalu tercakup.
{# Destinations #}

Destinations (Tujuan Backup)

Destinasi dikelola terpisah di menu Destinations, lalu dipilih per-project (satu project bisa beberapa destinasi sekaligus). Hasil backup otomatis dirapikan ke dalam sub-folder per project.

L Local

Salin arsip ke direktori pada server ini. Cukup isi path tujuan; folder dibuat otomatis bila belum ada.

S Amazon S3

Isi bucket, region, dan kredensial (Access Key & Secret). Mendukung endpoint S3-compatible (MinIO, Wasabi, dll). Bila prefix dikosongkan, arsip ditempatkan di quenza-backups/<project>/.

G Google Drive

Klik Connect Google Drive dan login dengan akun Anda (OAuth). Backup masuk ke Drive akun tersebut. Bila Folder ID dikosongkan, sistem membuat folder otomatis dan menyimpannya. Memerlukan GOOGLE_CLIENT_ID/SECRET & ENCRYPTION_KEY di server.

F FTP

Isi host, port (default 21), user, password, dan direktori tujuan. Direktori dibuat otomatis bila perlu.

C SCP / SSH

Transfer arsip ke server lain via SSH (SFTP). Mendukung autentikasi Password atau Private Key (tempel isi PEM atau path file, dengan passphrase opsional). Cocok untuk mengirim backup antar-server.

Tes Koneksi: setelah membuat destinasi, klik tombol Tes Koneksi pada kartunya untuk memastikan konfigurasi benar sebelum dipakai backup.
{# Schedule #}

Penjadwalan

Di detail project, aktifkan Penjadwalan dan pilih frekuensi:

  • • Harian — tiap hari pada jam tertentu.
  • • Mingguan — pada hari & jam tertentu.
  • • Bulanan — pada tanggal & jam tertentu.

Jam dijalankan mengikuti zona waktu global yang Anda atur di Settings. Lihat ringkasan semua jadwal di menu Schedules.

{# Manual backup #}

Backup Manual

Anda dapat menjalankan backup kapan saja tanpa menunggu jadwal:

  • • Tombol Run Backup di pojok kanan atas detail project.
  • • Tombol Backup Sekarang di kartu Penjadwalan.

Backup manual berjalan langsung dan tidak memengaruhi jadwal otomatis — keduanya berjalan independen. Hasil setiap eksekusi tercatat di History.

{# Restore #}

Restore

Restore bersifat pasif & aman: aplikasi hanya mengunduh arsip dari destinasi lalu mengekstraknya ke direktori tujuan. Tidak ada aksi otomatis (tanpa import database).

  1. 1. Buka menu Restore, pilih destinasi sumber.
  2. 2. Pilih arsip backup yang tersedia.
  3. 3. Tentukan direktori tujuan ekstraksi, lalu klik Jalankan Restore.
Ekstraksi dilindungi dari serangan path-traversal (zip-slip/tar-slip) — entri arsip yang mencoba keluar dari direktori tujuan akan ditolak.
{# Settings #}

Settings & Notifikasi

Zona Waktu — menentukan interpretasi jam jadwal dan tampilan waktu di seluruh aplikasi. Data tetap disimpan dalam UTC.

Notifikasi — pilih satu channel:

  • • Email — konfigurasi SMTP (host, port, user, password, pengirim) dan hingga 3 email penerima.
  • • Telegram — satu bot (token dari @BotFather) dan Chat ID tujuan.

Gunakan tombol Kirim Tes untuk memverifikasi konfigurasi. Notifikasi dikirim untuk setiap hasil backup & restore (atau hanya saat gagal, sesuai pilihan Anda).

{# Security #}

Keamanan

  • • Master Password disimpan sebagai hash bcrypt, tidak pernah plaintext.
  • • Kredensial destinasi (password FTP/SCP, secret key S3, token Drive, private key) dan kredensial notifikasi dienkripsi di database menggunakan kunci ENCRYPTION_KEY.
  • • Set DEBUG=false di produksi agar cookie session hanya dikirim via HTTPS.
  • • Restore dilindungi dari path-traversal.
